## Purpose
Alpaca Market Data v2 â†’ `List[Candle]`. Two-layer module: a pure response-mapper for offline tests and an HTTP fetcher gated on credentials.

## Public API
- `candles_from_alpaca_response(payload: dict, *, interval: str) -> List[Candle]` â€” pure mapper. Accepts either the standard envelope `{"bars": [...]}` or a bare list. Uses `candles_from_json_rows` with `ts_unit="iso"` (Alpaca returns ISO-8601 `t` values) **and `tz=core.timezones.ET`** so UTC bars are converted to US-Eastern wall-clock (matching yfinance); non-finite OHLC rows are skipped by that shared normalizer.
- `fetch_alpaca_data(ticker="AAPL", interval="1d", *, lookback_days=None, start=None, end=None) -> Optional[List[Candle]]` â€” `DataFetcher`-compatible. Without `start`/`end`, fetches a trailing window (`lookback_days` or `provider_lookback_days`). Passing kw-only `start`/`end` (aware datetimes) fetches that **explicit range** instead â€” the targeted intraday fetch path; this is what marks Alpaca `supports_range=True` at registration. Returns `None` on missing credentials, unsupported interval, or HTTP failure. Registered as `"alpaca"` in `DATA_SOURCES` (with `supports_range=True`) when `AlpacaCredentials.is_configured()`.

## Design Decisions
- **Static API-key auth** (no OAuth). Two headers â€” `APCA-API-KEY-ID` and `APCA-API-SECRET-KEY` â€” are set on each request.
- **Pagination via `next_page_token`**. `_http_get_page(...)` fetches one page (`limit=10000`, optional `page_token`); `_accumulate_bars(fetch_page)` walks pages until the payload has no `next_page_token`, concatenating every page's `bars` into one `{"bars": [...]}` envelope handed to the mapper. `fetch_page` is injected so the pagination loop is offline-testable. A `_MAX_PAGES=200` safety cap prevents an infinite loop if the vendor mis-paginates (never-null token); hitting it logs a WARNING and returns the truncated result. **We send `limit=10000` but the free IEX feed returns only ~2,000 bars/page in practice** (see below), so a wide window paginates: a 60-day 1-minute request silently truncated at the first page before `_accumulate_bars` existed.
- **IEX page size â‰ˆ 2,000 bars, NOT 10,000; round-trip cost is per-page (~0.6s)**, empirically verified. The API advertises a 10,000-bar `limit`, but the free-tier IEX historical feed caps each response at ~2,000 regardless. Round-trip time is dominated by the number of pages (~0.6s each), not the bar count within a page (79 bars = 0.32s, 1,732 bars = 0.64s â€” mostly fixed overhead). Consequence: `constants.DEFAULT_BARS_PER_PAGE` is **2,000**, so the targeted-fetch window (`targeted_window`) fills ~1 real page and a drilldown fetch is ~0.6s/symbol. Keeping `limit=10000` on the request is harmless and future-proofs a paid SIP feed (bigger pages) without a code change.
- **Rate limit: 200 API calls per MINUTE** on the free "Basic" plan (per Alpaca docs â€” NOT per second). Exceeding it returns HTTP 429. **Now retried** (perf item #4): `_http_get_page` routes through `_request_with_retry`, which retries 429 / 5xx (`_RETRYABLE_STATUS`) up to `_MAX_HTTP_RETRIES=3` times, honouring a `Retry-After` header when present (`_retry_after_seconds`) else exponential backoff (`0.5 * 2**attempt`, capped 8s); a non-retryable status or exhausted retries re-raises (â†’ caller's `None`). Additionally `_accumulate_bars` applies a light `_PAGE_PAUSE_S=0.2` throttle BETWEEN pages (never before the first) so a deep multi-page fetch doesn't burst the ceiling. Both `sleep_fn`s are injected so the retry/backoff/throttle policy is unit-tested offline with no real sleeps.
- **Proactive account-wide token bucket (the primary rate-limit tool).** For a *fixed* per-minute quota the right tool is proactive pacing, not reactive backoff (backoff is a poor fit when the recovery time is knowable). A single process-wide `data.rate_limiter.TokenBucket` (`_ALPACA_BUCKET`) paces ALL Alpaca requests â€” interactive fetches AND the universe preloader â€” since every call funnels through `_http_get_page`, which `.acquire()`s one token before spending an API call. The bucket rate is the tier budget: **free 200/min, paid 10,000/min** (`_ALPACA_RATE_BY_TIER`, `_alpaca_rate_per_min`). `_alpaca_bucket_for(creds)` reconfigures the shared bucket live when the tier changes (freeâ†”paid), so an upgrade takes effect without a restart. The bucket keeps a 10% safety margin (sustained ~90% of the limit + a burst = the remaining headroom) so a rolling-minute never exceeds the quota; the `Retry-After` retry above is now only a rare-overshoot net (e.g. a mis-set "paid" on a free key).
- **Header-driven free-tier auto-detect + downgrade.** Alpaca returns `X-RateLimit-Limit` on every data response (200 free / 10,000 paid); a free key sending `feed=sip` 403s but the 403 still carries it. `_observe_rate_limit_header(headers)` runs on both the success path (`_do_request`) and the error path (`_request_with_retry`). If it reveals the free budget (â‰¤200) while the bucket was pacing for paid, it **downgrades once (downward-only)**: latches `_detected_free`, clamps the bucket to 200, and â€” via `_alpaca_rate_per_min` capping + a `feed="iex"` override in `_http_get_page` â€” makes the downgrade stick even against a persisted `tier="paid"` (and stops the `feed=sip` 403 loop). It records a **one-shot** popup notice; `pop_pending_downgrade_notice()` is polled on the Tk thread by `gui/polling._drain_worker_inbox` (cross-thread Tk from the fetch worker is unsafe on this build) and shown as a `messagebox.showerror`. Never auto-*raises* a selected plan. `_reset_tier_detection()` is a test-only reset for the module-global latch.
- **Symbol translation `_to_alpaca_symbol` (perf item #3)**: the app normalizes every symbol to yfinance's **dash** share-class form (`BRK-B`), but Alpaca's stocks API expects a **dot** (`BRK.B`). `fetch_alpaca_data` converts via `_to_alpaca_symbol` (uppercase + `-`â†’`.`) before the HTTP call â€” previously the dash form silently returned empty for every share-class ticker. Plain symbols (`AMD`) pass through unchanged; ratio pseudo-symbols (`/`) never reach here.
- **Shared `credentialed_opener()`** (security audit I4 / M5). HTTP
  call routed through `data._http.credentialed_opener` so cross-host
  30x redirects strip both `APCA-API-KEY-ID` and `APCA-API-SECRET-KEY`
  before forwarding. The response read is bounded by
  `data._http.MAX_RESPONSE_BYTES` (8 MB) to defend against
  pathological server replies.
- **`feed` is part of credentials, defaults to `"iex"`**: free-tier feed. Configured via `ALPACA_FEED` env var if a paid SIP subscription is in play.
- **`adjustment` is configurable, default `split`** (perf item #2): sent to the `/bars` endpoint via `_resolve_adjustment(creds)`, which validates `creds.adjustment` against `{raw, split, dividend, all}` and falls back to `split` (with a warning) on an unknown value. **Default changed from `raw` â†’ `split`** so a post-split chart doesn't render a fake 90%-crash cliff (NVDA/AAPL etc.) â€” jarring next to yfinance's auto-adjusted series. `split` back-adjusts splits only (price levels intact); set `ALPACA_ADJUSTMENT=all` for yfinance-`auto_adjust`-equivalent split+dividend adjustment, or `raw` for un-adjusted. The value lives on `AlpacaCredentials.adjustment` (resolved from the `ALPACA_ADJUSTMENT` credential/env var).
- **Default lookback** via `constants.provider_lookback_days("alpaca", interval)`: Alpaca has no yfinance 60-day intraday cap, but each intraday window is **fetch-speed-bounded** (the whole series loads up front, so 5m â‰ˆ 4 months / ~120d, 1h â‰ˆ 4y, 1m â‰ˆ 1mo â€” each ~1 API page / â‰²3s to clear the 5s drilldown deadline). Daily requests ~15y (the server caps to the plan's availability â€” free IEX â‰ˆ 6y for AAPL). Replaces the old 730d/60d yfinance-matched cap that truncated daily history to ~2 years and drilldown to ~60 days.
- **Interval map**: `{1m,5m,15m,30m,1h,1d,1wk,1mo} â†’ Alpaca's `"1Min" / "1Hour" / "1Day"` etc.`
- **Non-finite OHLC rows are dropped by the shared normalizer** before `Candle` construction.
- **Never raises**: HTTP/JSON errors caught broadly; logged at WARNING; returns `None`.

## Invariants
- Returns either `None` or a (possibly empty) list of `Candle`. Never raises.
- Timestamps arrive UTC (ISO with `Z` suffix) and are converted to **US Eastern** (`core.timezones.ET`) by the mapper, so `classify_session` and the chart read correct exchange wall-clock. Without this a 09:30 ET open bar (14:30Z) is mis-classified and the intraday session is shifted +5h (the "5m data only shows 14:30â€“16:00" bug). Falls back to UTC if `tzdata`/`ET` is unavailable.
- The interval keyspace matches `_INTERVAL_TO_ALPACA`; other intervals are rejected before HTTP.
